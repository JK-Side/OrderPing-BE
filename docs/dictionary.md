## PRD — 용어 사전(Glossary) v0.3

이 문서는 오더핑 PRD/설계/개발 전 과정에서 같은 단어를 같은 의미로 쓰기 위한 기준이다.  
**UI 용어**와 **도메인(데이터/상태) 용어**를 구분하고, 혼선이 잦은 단어는 더 작은 개념으로 분리한다.

---

### 1) 사용자 / 계정

| 용어 | 정의 | 범위/비고 |
| --- | --- | --- |
| 운영자(Operator) | 오더핑에 로그인해 주점 운영 기능을 사용하는 주체 | 로그인 필요 |
| 주문자(Guest / Orderer) | QR을 통해 주문하는 주점 손님 | 비로그인(세션 기반) |
| 계정(Account) | 운영자 로그인 단위(아이디/비번 또는 OAuth 식별자) | 인증/인가의 기준 |
| 마이페이지(My Page) | 운영자 개인 설정 및 계정/주점 목록 관리 화면 | 주점 설정과 분리 |

---

### 2) 주점(Store)

| 용어 | 정의 | 범위/비고 |
| --- | --- | --- |
| 주점(Store) | 운영자가 운영하는 하나의 판매 단위 공간 | 운영 단위(정산/메뉴/테이블의 기준) |
| 주점 카드(Store Card) | 주점을 대표하는 썸네일 + 이름 UI 요소 | UI 용어 |
| 주점 설정(Store Settings) | 주점 기본 정보/결제 정보/메뉴/테이블 등의 설정 변경 영역 | 운영(Operation)과 구분 |
| 주점 운영(Operation) | 주문 조회/테이블 관리/통계 등 실시간 운영 영역 | 실시간 처리 중심 |
| 주점 수정(Edit Store) | 주점명/설명/이미지 등 기본 정보 수정 | 설정 내 하위 기능 |

---

### 3) 온보딩(Onboarding)

| 용어 | 정의 | 범위/비고 |
| --- | --- | --- |
| 온보딩(Onboarding) | 신규 운영자가 최초로 주점 정보와 결제 정보를 입력해 “주점 생성”을 완료하는 흐름 | 최초 1회가 기본 |
| Step 1 — 주점 정보 등록 | 주점명 / 설명 / 대표 이미지 입력 단계 | 필수/선택 항목은 별도 정의 |
| Step 2 — 계좌 정보 등록 | 정산 계좌 정보 및 결제 QR 이미지 등록 단계 | QR은 업로드 파일 |
| Step 3 — 주점 시작 | 온보딩 완료 후 운영 화면 진입 안내 단계 | 실제 영업 상태 변경과 분리 여부는 정책(결정 필요) |
| 스텝 인디케이터(Step Indicator) | 온보딩 진행 상태를 표시하는 UI 요소 | UI 용어 |

---

### 4) 결제 / 정산

| 용어 | 정의 | 범위/비고 |
| --- | --- | --- |
| 정산 계좌(Settlement Account) | 손님 입금 안내 및 매출 정산 기준으로 사용되는 계좌 정보 | 주점 단위 1개 기본 |
| 은행명(Bank Name) | 계좌가 개설된 은행 이름 | 표준화는 후순위 가능 |
| 예금주(Account Holder) | 계좌 실소유자 이름 | 검증/인증은 MVP 제외 권장 |
| 계좌번호(Account Number) | 정산금이 입금되는 계좌 번호 | 화면 표시 시 마스킹 필요 |
| 결제 QR 코드(Payment QR) | 손님 결제를 유도하기 위해 등록하는 QR 이미지 | “토스 QR” 종속을 피하려면 일반화 권장 |
| 결제 타입(Payment Type) | 결제 수단의 분류(예: 현금/쿠폰) | 통계 분리의 기준 |
| 현금(CASH) | 계좌이체 등 현금성 결제 | 결제 타입 |
| 쿠폰(COUPON) | 쿠폰으로 대체 결제 | 결제 타입 |
| 정산(Settlement) | 마감 시점에 결제 완료 주문을 기준으로 매출/판매량을 집계하는 행위 | “결제 안내”와 구분 필요 |

---

### 5) 주문(도메인) / 주문 처리 상태

| 용어 | 정의 | 범위/비고 |
| --- | --- | --- |
| 주문(Order) | 특정 테이블에서 생성된 주문 단위(여러 주문 항목 포함) | 핵심 도메인 |
| 주문 항목(Order Item) | 주문 내 개별 상품/수량/가격 스냅샷 | 가격 변동 대비해 스냅샷 저장 권장 |
| 주문 번호(Order No.) | 주점 내에서 순차 부여되는 사람이 읽기 쉬운 주문 식별 번호 | DB ID와 구분 |
| 주문 상태(Order Status) | 주문 처리 진행 상태를 나타내는 값 | 운영 큐 구성의 기준 |
| 결제 확인 전(Payment Pending) | 주문은 접수되었으나 운영자가 결제를 확인하지 않은 상태 | 기존 용어 유지 |
| 조리 중(Preparing) | 결제 확인이 완료되어 조리/준비 단계인 상태 | “결제 완료”와 화면 표기는 분리 가능 |
| 서빙 완료(Served) | 주문 제공 완료 상태 | 최종 상태 |
| 주문 카드(Order Card) | 운영 화면에서 주문을 요약해 보여주는 카드형 UI 요소 | UI 용어(주문번호/테이블/총액/상태 등) |

> 참고: “입금자명/입금금액”은 주문의 본질 정보라기보다 결제 확인을 돕는 정보다. 필요하다면 **결제 요청(Payment Request)** 개념을 별도로 두는 것이 모델/감사 로그에 유리하다(추가 여부는 정책으로 확정).

---

### 6) 테이블 / 좌석

| 용어 | 정의 | 범위/비고 |
| --- | --- | --- |
| 테이블(Table) | 주문을 귀속시키는 좌석 식별 단위 | QR 연결의 기준 |
| 테이블 번호(Table No.) | 주점 내 테이블을 구분하는 고유 번호 | 주점 내 유니크 |
| 테이블 카드(Table Card) | 운영 화면에서 테이블 상태/잔여 시간/주문 요약을 보여주는 UI 요소 | UI 용어 |
| 테이블 QR(Table QR) | 손님이 해당 테이블로 주문하기 위해 스캔하는 QR | 테이블 식별값 포함 |
| 서비스 추가(Add Service) | 기존 테이블에서 추가 주문을 생성하는 행위 | 구현상 “새 주문 생성”으로 처리하는 것이 단순 |

---

### 7) 통계 / 데이터(백로그 포함)

| 용어 | 정의 | 범위/비고 |
| --- | --- | --- |
| 매출(Sales) | 결제 확인 완료(조리 중/서빙 완료 포함) 주문을 기준으로 합산한 금액 | 기준 상태를 명확히 고정 |
| 결제 타입별 매출 | 현금/쿠폰 등 결제 타입별 매출 합계 | 운영 요구사항(분리 필요) |
| 판매량(Sales Volume) | 상품별 판매 수량 합계 | Order Item 합산 |
| 전체 주문 조회 | 특정 기간/상태 조건으로 주문을 조회하는 기능 | 통계/감사 용도 |
| 시간대별 매출(Hourly Sales) | 시간 구간별 매출 합계 | 백로그 |

---

## 용어 사전 적용 규칙
- 문서/코드에서 “User”는 가급적 **운영자(Operator)** 로 표기한다(주문자와 혼선 방지).
- “정산”은 마감 집계 의미로 사용하고, 손님에게 보여주는 것은 “결제 안내/결제 정보”로 분리한다.
- “주문 번호(Order No.)”와 “주문 ID(DB PK)”를 반드시 구분한다(운영 화면에는 Order No. 중심).
